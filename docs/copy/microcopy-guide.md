# Microcopy Guide

> Generated by forge-copy v1.0.0 | 2026-03-12
> Brand voice: прямой, практичный, дружелюбный, уверенный, экспертный

## Voice Summary

Говорим прямо, без воды и маркетингового шума. Каждое слово несёт пользу. Общаемся как с коллегой — дружелюбно, но уверенно. Экспертность проявляется через конкретику, не через громкие слова.

## Word List

| ✅ Использовать | ❌ Избегать |
|----------------|------------|
| продукт | ПО, программное обеспечение |
| запустить | имплементировать, реализовать |
| под ключ | комплексное решение |
| идея → реальность | инновационный подход |
| современный стек | передовые технологии |
| быстро | в кратчайшие сроки |
| предприниматель, founder | клиент, заказчик |
| работает | функционирует |

## Patterns by Context

### Actions (кнопки, ссылки, CTA)

| Контекст | Текст (RU) | Текст (EN) | Заметки |
|----------|-----------|-----------|---------|
| Главный CTA | Связаться со мной | Get in touch | Первый по порядку, самый заметный |
| Вторичный CTA | Смотреть проекты | View projects | Менее акцентный |
| Отправка формы | Отправить заявку | Send request | Конкретное действие, не "Отправить" |
| Ссылка на демо | Демо | Demo | Кратко |
| Ссылка на код | GitHub | GitHub | Без перевода |

**Правило:** Кнопки = глагол + объект. "Отправить заявку", не "Отправить". "Смотреть проекты", не "Проекты".

### Feedback (сообщения об успехе, ошибках, загрузке)

#### Успех

| Контекст | Текст (RU) | Текст (EN) |
|----------|-----------|-----------|
| Форма отправлена | Заявка отправлена! | Request sent! |
| Подтверждение | Спасибо за обращение. Я свяжусь с вами в ближайшее время. | Thank you for reaching out. I'll get back to you shortly. |

**Правило:** Успех = что произошло + что будет дальше.

#### Ошибки

| Контекст | Текст (RU) | Текст (EN) |
|----------|-----------|-----------|
| Общая ошибка | Что-то пошло не так | Something went wrong |
| Сеть | Ошибка сети. Попробуйте позже. | Network error. Please try again later. |
| Rate limit | Подождите 30 секунд перед повторной отправкой | Please wait 30 seconds before resubmitting |
| Невалидные данные | Проверьте введённые данные | Please check your input |
| Сервер | Не удалось отправить сообщение | Failed to send message |

**Правило:** Ошибки = что не так + что делать. Без технических деталей. Без восклицательных знаков.

### Forms (метки, плейсхолдеры, валидация)

#### Метки и плейсхолдеры

| Поле | Метка (RU) | Плейсхолдер (RU) |
|------|-----------|-----------------|
| Имя | Имя | Ваше имя |
| Контакт | Контакт | Email или @username в Telegram |
| Сообщение | Сообщение | Расскажите о вашем проекте или задаче... |

**Правило:** Плейсхолдеры — подсказка, что вводить. Не дублируют метку. Формат через пример.

#### Валидация

| Правило | Текст (RU) | Текст (EN) |
|---------|-----------|-----------|
| Имя < 2 символов | Имя должно содержать минимум 2 символа | Name must be at least 2 characters |
| Имя слишком длинное | Имя слишком длинное | Name is too long |
| Контакт пустой | Укажите email или Telegram | Please provide email or Telegram |
| Контакт слишком длинный | Контакт слишком длинный | Contact is too long |
| Сообщение < 10 символов | Сообщение должно содержать минимум 10 символов | Message must be at least 10 characters |
| Сообщение слишком длинное | Сообщение слишком длинное | Message is too long |

**Правило:** Валидация = что не так + как исправить. "Минимум 2 символа", а не "Ошибка валидации поля name".

### Empty States

Не применимо — портфолио не имеет пустых состояний для пользователя.

### Navigation

| Элемент | Текст (RU) | Текст (EN) |
|---------|-----------|-----------|
| О себе | Обо мне | About |
| Навыки | Навыки | Skills |
| Проекты | Проекты | Projects |
| Контакт | Связаться | Contact |
| Язык | RU / EN | EN / RU |

**Правило:** Навигация — 1 слово, без глаголов. Исключение: "Связаться" — это CTA в навигации.

### Headings (заголовки секций)

| Секция | Заголовок (RU) | Подзаголовок (RU) |
|--------|---------------|-------------------|
| Hero | — | Создаю современные веб-приложения от идеи до продакшена |
| About | Обо мне | Немного о моём опыте и подходе к разработке |
| Skills | Навыки | Технологии, с которыми я работаю каждый день |
| Projects | Проекты | Избранные работы из моего портфолио |
| Contact | Связаться | Есть проект или идея? Напишите мне! |

**Правило:** Заголовок = тема (1-2 слова). Подзаголовок = конкретика или call-to-action.

## Tone Checklist

Перед публикацией любого текста:
- [ ] Понятно ли с первого прочтения?
- [ ] Нет ли маркетингового шума (инновационный, уникальный)?
- [ ] Конкретно ли? (цифры, примеры вместо абстракций)
- [ ] Дружелюбно, но не панибратски?
- [ ] Одинаковые термины для одинаковых вещей?
